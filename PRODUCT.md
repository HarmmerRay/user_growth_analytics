# PRODUCT — 用户增长分析

## 产品定位

User Growth Analytics 是一款基于 Frappe 的用户增长分析应用。  
以「用户服务开通 / 流失事件」为单一数据源，提供列表录入、增长报表与投屏大屏，帮助团队观察获客、流失与 MRR 变化。

## 产品形态

| 形态 | 说明 |
|------|------|
| Desk 单据 | 业务人员录入或导入开通 / 流失事件 |
| Desk 报表 | 分析人员按时间与维度查看增长质量 |
| Desk 大屏 | 适合会议室 / 运营大屏的只读可视化看板 |

三者共用同一 DocType，不另建平行事实表。

## 功能清单

### 1. 用户服务事件（User Service Event）

记录每一次开通或流失：

- 用户标识与姓名
- 事件类型：开通 / 流失
- 事件日期
- 服务套餐、地区、获客渠道、终端
- 月费金额（MRR）
- 备注

命名规则：`USE-.YYYY.-.#####`。  
安装后可自动写入近 12 个月示例数据，便于开箱查看报表与大屏。

### 2. 用户增长报表（User Growth Summary）

基于事件表按月聚合：

- 开通数、流失数、净增长
- 期末活跃（估算）
- 流失率
- 新增 / 流失 / 净增 MRR

支持筛选：日期范围、地区、套餐。  
提供折线图与顶部汇总指标。

入口：`/app/query-report/User%20Growth%20Summary`

### 3. 用户增长数据大屏（user-growth-dashboard）

投屏向只读看板：

- KPI：累计开通 / 流失 / 净增长、期末活跃、流失率、净增 MRR
- 图表：月度趋势、活跃走势、地区分布、渠道结构、套餐结构
- 约 60 秒自动刷新

入口：`/app/user-growth-dashboard`

## 核心用户与场景

| 角色 | 场景 |
|------|------|
| 运营 / 增长 | 看大屏掌握整体走势与渠道结构 |
| 分析 | 用报表按月、按地区拆增长与流失 |
| 业务录入 | 在单据中维护开通 / 流失事实 |

## 非目标（当前版本不做）

- 不替代完整 CRM / 计费系统
- 不做多租户 SaaS 计费编排
- 不依赖 ERPNext 模块
- 不做移动端独立 App（使用 Frappe Desk）

## 成功标准

1. 安装 App 后即可看到示例数据、报表与大屏。  
2. 单据变更后，报表与大屏聚合结果一致。  
3. 仅依赖 Frappe + Docker 即可本地复现。
